# Support

Leanforge support is handled through GitHub repository workflows.

## Before asking for help

Check these docs first:

- [Installation](docs/installation.md)
- [Troubleshooting](docs/troubleshooting.md)
- [Dryforge to Leanforge migration](docs/migration-dryforge-to-leanforge.md)
- [Examples](examples/README.md)

## Good support request

Include:

- client: Claude Code or Codex;
- Leanforge version or tag, if known;
- operating system;
- command used, such as `/leanforge:prime`, `/leanforge:run`, or
  `/leanforge:set`;
- what you expected;
- what happened instead;
- relevant logs or screenshots with secrets removed;
- whether `.leanforge/` or legacy `.dryforge/` state exists.

## Where to ask

Use GitHub issues for reproducible bugs, documentation gaps, and feature
requests. Keep each issue focused on one problem.

## What not to share

Do not share:

- secrets, tokens, API keys, or private credentials;
- proprietary source code that you are not allowed to disclose;
- private customer data;
- full `.leanforge/` or `.dryforge/` state if it contains sensitive project
  details.

## Security issues

Do not open a public support issue for suspected vulnerabilities. Follow
[SECURITY.md](SECURITY.md).
