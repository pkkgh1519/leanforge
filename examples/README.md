# Examples

Leanforge examples are copy-paste scenarios for understanding the workflow. They
are not generated applications and do not define product requirements for your
project.

| Example | Shows |
|---|---|
| [Booking system](booking-system/README.md) | Intent elicitation, acceptance evidence, and first-run expectations for a small behavior-changing feature. |

Use examples when you want to see what a good Leanforge input looks like before
running `Leanforge:Prime` in your own repository.
